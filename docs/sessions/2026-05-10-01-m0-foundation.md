## What landed

### `src/Simulation/Core/FixedMath.cs`
- **Sqrt**: shift-and-subtract integer sqrt of `raw << 16` using `UInt128`. Deterministic, no Newton iteration convergence ambiguity, returns floor of the true square root in Q.16.
- **Sin / Cos**: argument reduction to `(-π, π]` via modulo on raw, then to `[-π/2, π/2]` via `sin(π - x) = sin(x)`, then a 4-term Taylor in Horner form. Coefficients `1/6 = 10923`, `1/120 = 546`, `1/5040 = 13` (higher terms are zero at Q.16 precision). `Cos(x) = Sin(x + π/2)`.
- **Atan2**: quadrant fold + `|y|/|x|` swap to bring the ratio into `[0, 1]`, then the half-angle reduction `atan(r) = π/4 + atan((r-1)/(r+1))` to bring it into `[0, tan(π/8)]`, then a 5-term Taylor Horner.
- Tightened `Pi/TwoPi/HalfPi` to be internally consistent (`TwoPi.Raw == 2 * Pi.Raw`, `HalfPi.Raw == Pi.Raw / 2 + 1`).

Tests in `tests/Simulation.Tests/FixedMathTests.cs` cover:
- `Sqrt`: pinned raw values for `Sqrt(2)`, `Sqrt(3)`, `Sqrt(0.5)`; exactness for perfect squares; range check for `Sqrt(10^9)`; throws on negative.
- `Sin / Cos`: cardinal-angle assertions within 50-raw tolerance (~7.6e-4); `sin²+cos² ≈ 1` invariant at 16 angles around the full circle within 200 raw.
- `Atan2`: exact returns at the four cardinal axes; `atan2(1,1) ≈ π/4` and `atan2(-1,-1) ≈ -3π/4` within tolerance; round-trip `atan2(sin a, cos a) ≈ a` across 8 directions, skipping the `-π` branch cut.

### `src/Simulation/Core/Fnv1a64.cs` + `IHashable.cs`
FNV-1a 64-bit streaming hash with `Mix(ulong|long|uint|int)` byte-order-explicit overloads. Chosen over xxHash64 because the brief allowed either and FNV-1a needs no NuGet package. `IHashable` is the contract entities will adopt in M1.

### `src/Simulation/World.cs` — `ComputeStateHash`
Hashes `CurrentTick`, `_nextEntityId`, the RNG's four state words (via `DeterministicRng.HashInto`), then iterates the sorted entity dictionary in `EntityId` order, mixing each ID plus `HashInto` if the entity implements `IHashable`. No-op-for-now on entity payloads since none exist yet, but the pattern is correct.

Tests in `tests/Simulation.Tests/WorldHashTests.cs`:
- Same seed → same hash.
- Different seeds → different hash.
- Stepping the world changes the hash.
- Allocating an `EntityId` changes the hash even before registration (since `_nextEntityId` is hashed).
- Hash is non-zero on an empty world.

### `scripts/check-sim-purity.sh`
Bash script greps `src/Simulation/` for `using Godot;`, bare `float`/`double` declarations, `System.Random`, `DateTime`, `Stopwatch`, `Environment.TickCount`, `Math.*`, `Guid.NewGuid`, `async`/`await`/`Task.Run`/`Parallel.*`. Skips comment lines so docstrings mentioning the forbidden names don't false-positive. Allows the sanctioned `ToFloatForRender()` exception in `Fixed64.cs`.

Verified by injecting `using Godot;` into `World.cs`, confirming the script exits 1, then removing it and confirming exit 0.

### `.github/workflows/ci.yml`
New `sim-purity` job runs first on every PR (ubuntu-latest). The matrix `test` job now `needs: sim-purity`, so a purity violation short-circuits the whole pipeline.
